# Code of Conduct

## The rules

1. **Technical disagreements are welcome.** Be specific about what's wrong and
   why. Bring evidence — benchmarks, reproductions, citations.

2. **Personal attacks, harassment, and discrimination on any grounds — race,
   gender, sexual orientation, religion, nationality, disability, age — are
   not tolerated.** This applies in issues, pull requests, discussions, code
   comments, commit messages, and any project-related communication channel.

3. **No spam, no off-topic promotional content, no recruitment outreach in
   technical threads.** Use direct contact for those.

4. **Disagree without being a jerk.** "Your code is wrong, here's why" is fine;
   "you are stupid" is not.

## Enforcement

Violations result in one or more of: comment removal, temporary suspension,
or permanent ban from the project's communication channels.

Report violations to **info@macstab.com** with subject prefix `[CONDUCT]`.
Reports are read by the project maintainer (Christian Schnapka), kept
confidential, and acted on within **7 days**.

This project's standards align with the
[Contributor Covenant 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/);
that text governs in any case the rules above leave ambiguous.
