# Code of Conduct

This project welcomes anyone who wants to make it better. Submit a bug
report, ask a question, write a patch, push back on a design — the door
is open.

We expect everyone in this community to behave like a thoughtful adult.
The rules below exist to make that explicit, not because we expect to need
them.

## The rules

1. **Technical disagreements are welcome.** Be specific about what's wrong
   and why. Bring evidence — benchmarks, reproductions, citations, code.
   *"This is slow"* is the start of a conversation; *"This is slow because
   we allocate inside the loop, see profile attached"* is the conversation.

2. **Personal attacks, harassment, and discrimination on any grounds —
   race, gender, sexual orientation, religion, nationality, disability,
   age, or any other personal characteristic — are not tolerated.** This
   applies in issues, pull requests, discussions, code, commit messages,
   and any other project-related communication.

3. **No spam, no off-topic promotional content, no recruitment outreach
   in technical threads.** Use direct contact for those.

4. **Disagree without being a jerk.** *"Your code is wrong, here's why"*
   is fine. *"You are stupid"* is not. The first one leads to better
   software. The second one leads to a smaller community.

## Where these rules apply

Everywhere this project lives — GitHub issues and pull requests, GitHub
Discussions, the project's website and blog comments, any chat channel
operated by the project, and conferences or meetups where you're
representing this project.

Anywhere else — your personal accounts, your blog, your dinner table —
is yours. We hope you live by these rules anyway.

## Enforcement

Violations result in one or more of: comment removal, temporary suspension,
or permanent ban from the project's communication channels. Severity,
repetition, and remorse all factor in.

Report violations to **info@macstab.com** with subject prefix `[CONDUCT]`.
Reports are read by the project maintainer (**Macstab GmbH / Christian
Schnapka**), kept confidential, and acted on within **7 days**.

If the maintainer is the person you need to report, contact us via
[macstab.com](https://macstab.com) — the message will be routed to an
independent reviewer.

## Standards alignment

This project's standards align with the
[Contributor Covenant 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/).
That text governs anywhere the rules above leave ambiguous.
